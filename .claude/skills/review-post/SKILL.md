# 포스트 품질 검증

## 입력
- `$ARGUMENTS`: 검증할 포스트 파일 경로 (예: _posts/2026-04-02-example.md)

## 검증 항목

### 1. Front Matter 완성도
- title, date, categories, tags, excerpt 모두 존재하는가
- categories가 유효한 슬러그인가 (6개 중 하나)
- excerpt가 100자 이내인가

### 2. 블랙리스트 검사
- `.claude/rules/blacklist.md`의 금지 표현이 본문에 있는가
- 빈 수식어가 구체적 수치 없이 사용되었는가

### 3. 소스 정책
- 근거 없는 주장이 있는가
- Tier 3 소스를 인용하고 있는가
- 링크 텍스트가 서술적인가 ("여기" "링크" 금지)

### 4. 톤앤매너
- ~다 체를 일관되게 사용하는가
- 문장 길이가 40자를 크게 초과하는가
- AI 특유 표현이 있는가

### 5. 구조/분량
- 분량이 1,500~3,000자 범위인가
- 서두(왜)/본문(과정)/결론(배운 점) 구조가 있는가

## 출력 형식
```
## 검증 결과: [합격/불합격]

### 발견된 문제
- [심각] ...
- [경고] ...
- [참고] ...

### 수정 제안
1. ...
2. ...
```

## 중요
- 파일을 수정하지 않는다. 피드백만 출력한다.
