기업에서 AI를 도입할수록 데이터 품질 문제가 수면 위로 올라온다. 입력 데이터가 엉망이면 모델 결과도 틀린다는 건 알지만, 어떤 기준으로 관리해야 하는지 모호한 경우가 많다. 이 글은 데이터 거버넌스의 국제 표준 프레임워크인 DAMA-DMBOK을 중심으로 2026년 시점의 과제를 정리한다.

---

## 데이터 거버넌스란 무엇인가

데이터 거버넌스(Data Governance)는 데이터의 수집·저장·활용 전 과정에 정책·역할·표준을 적용하는 관리 체계다. 단순히 DB를 잘 관리하는 것이 아니라, 누가 어떤 데이터를 어떤 기준으로 다룰지를 조직 전체에서 정의하는 작업이다.

시장 규모로 보면 이 분야가 얼마나 급성장하고 있는지 알 수 있다. Fortune Business Insights에 따르면 전 세계 데이터 거버넌스 시장은 2025년 기준 약 53.8억 달러다. 2034년까지 연평균 20.5% 성장해 240.7억 달러에 이른다.[^1]

중요성에 대한 인식도 높아졌다. CIO Korea에 따르면 데이터 리더 다수가 거버넌스를 전략적 우선순위로 꼽는다.[^6] 그러나 Gartner는 "2027년까지 거버넌스 이니셔티브의 80%가 실패할 것"이라고 예측했다.[^6] 도구가 없어서가 아니라, 조직이 유지하지 못해서 실패한다는 의미다.

---

## DAMA-DMBOK 프레임워크

DAMA-DMBOK(Data Management Body of Knowledge)은 DAMA International이 발행하는 데이터 관리 표준 지식 체계다. DAMA International은 데이터 관리 전문가 협회로, DMBOK은 전 세계 데이터 관리 실무의 기준서 역할을 한다.[^3]

프레임워크는 11개 핵심 영역으로 구성된다. 데이터 거버넌스가 중심 허브이고, 나머지 10개가 그것에 연결된 구조다.

| 영역 | 주요 내용 |
|------|-----------|
| 데이터 거버넌스 | 정책·역할·표준 정의 (중심 허브) |
| 데이터 아키텍처 | 전사 데이터 구조 설계 |
| 데이터 모델링·설계 | 논리·물리 모델 정의 |
| 데이터 저장·운영 | DB 관리, 백업, 성능 |
| 데이터 보안 | 접근 제어, 개인정보 보호 |
| 데이터 통합·상호운용성 | ETL, API, 데이터 파이프라인 |
| 문서·콘텐츠 관리 | 비정형 데이터 관리 |
| 참조·마스터 데이터 | 공통 코드, 고객 마스터 등 |
| 데이터 웨어하우스·BI | 분석용 데이터 구조 |
| 메타데이터 관리 | 데이터에 대한 데이터 |
| 데이터 품질 관리 | 품질 측정·개선 |

현재 DMBOK은 2.0 버전이 현행 기준이며, 3.0 현대화 프로젝트가 2025년 착수됐다. DAMA International이 공개한 일정에 따르면 2025년 6월 25일 킥오프를 시작으로 AI, 클라우드, 현대 데이터 플랫폼을 반영한 커뮤니티 주도 업데이트가 진행 중이다.[^4]

## AI 시대에 거버넌스가 중요해진 3가지 이유

### 1. AI 결과는 입력 데이터 품질에 직결된다

AI 모델은 학습 데이터가 틀리면 결과도 틀린다. 문제는 이 오류가 눈에 잘 안 보인다는 점이다. 낮은 품질의 데이터는 잘못된 예측을 만들고, 그 결과가 의사결정에 반영되면 연쇄 오류로 이어진다. [삼성SDS 인사이트](https://www.samsungsds.com/kr/insights/data-management-trends-2026.html)도 2026년 데이터 관리의 핵심을 "확산보다 품질과 통제"로 정리한다.

### 2. 규제 대응이 거버넌스 의무를 만들었다

EU AI Act는 고위험 AI 시스템에 대해 학습 데이터의 품질 관리와 편향(bias) 검토를 요구한다. 한국 AI 기본법도 고영향 AI에 대해 데이터 거버넌스 관련 의무를 담고 있다 ([국가법령정보센터](https://www.law.go.kr/lsInfoP.do?lsiSeq=268543)). AI를 쓰는 기업은 이제 데이터 거버넌스를 선택이 아닌 컴플라이언스 요건으로 다뤄야 한다.

### 3. 설명 가능성에는 데이터 계보가 필요하다

데이터 계보(Data Lineage)는 데이터가 어디서 왔고 어떤 변환을 거쳤는지 추적하는 기능이다. AI 모델이 왜 그 결과를 냈는지 설명하려면 입력 데이터의 출처와 처리 과정이 기록돼 있어야 한다. 설명 가능한 AI(XAI)의 전제 조건이 데이터 계보 관리다.

## 데이터 품질 6가지 기준 (DMBOK 기준)

DMBOK은 데이터 품질을 6가지 기준으로 측정한다.[^3]

1. **정확성(Accuracy)** — 실제 값과 저장 값이 일치하는가
2. **완전성(Completeness)** — 필수 필드에 누락이 없는가
3. **일관성(Consistency)** — 시스템 간 동일 데이터가 같은 값을 가지는가
4. **적시성(Timeliness)** — 필요한 시점에 최신 데이터가 제공되는가
5. **유효성(Validity)** — 정의된 형식과 범위 안에 있는가
6. **고유성(Uniqueness)** — 중복 레코드 없이 단일하게 존재하는가

6가지 중 하나라도 무너지면 AI 학습 결과는 신뢰하기 어렵다. 예를 들어 고객 테이블에 동일 고객이 3개 레코드로 분리돼 있으면 고유성 기준을 위반한 것이고, 이를 기반으로 학습한 추천 모델은 같은 고객에게 중복 마케팅을 보내게 된다.

## 실무 대응 3가지

### 데이터 스튜어드 지정

데이터 스튜어드(Data Steward)는 부서별 데이터 책임자다. 전사 데이터 정책을 실제 현업에 연결하는 역할이다. 스튜어드가 없으면 거버넌스 정책은 문서로만 존재하게 된다.

### 데이터 카탈로그 도입

데이터 카탈로그는 조직 내 데이터 자산의 메타데이터를 한 곳에서 검색·관리하는 도구다. "고객 ID가 어느 테이블에 있는지, 누가 관리하는지"를 카탈로그에서 확인한다. 이것이 데이터 계보 추적의 출발점이다.

### 품질 대시보드 운영

6가지 기준을 지표로 만들어 실시간 모니터링한다. 기준별 점수가 임계값 아래로 내려가면 알림이 발생하고 담당 스튜어드가 조치한다. 대시보드 없이 주기적으로 수동 점검하는 방식은 AI 운영 환경에서 속도를 맞추기 어렵다.

## 정리

자료를 정리하면서 확인한 점은, 거버넌스는 도구가 아니라 조직 설계의 문제라는 것이다. 역할 정의와 프로세스 설계가 기술 도입보다 앞서야 한다. Gartner 예측처럼 실패의 원인이 기술이 아닌 채택에 있다면, 도구보다 사람과 프로세스를 먼저 정비해야 한다.

다음 단계로는 DMBOK 3.0 공개 초안이 나오는 시점에 AI 거버넌스 영역이 어떻게 통합되는지 추적할 예정이다. DAMA International의 공개 일정에 따르면 분기별 타운홀에서 진행 상황을 공개한다.[^4]
