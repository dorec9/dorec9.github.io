A/B테스트를 공부하면서 가장 먼저 부딪힌 질문이 "표본은 몇 명이나 모아야 하는가"였다. 답을 찾아보니 단순한 숫자 문제가 아니라, 설계 단계에서 결정해야 할 변수가 여러 개 얽혀 있었다. 이 글에서는 표본 크기 계산에서 자주 발생하는 5가지 실수와 올바른 계산법을 정리한다.

---

## 표본 크기가 왜 중요한가

A/B테스트의 목적은 두 그룹 간 차이가 우연인지 실제 효과인지 판별하는 것이다. 표본이 너무 작으면 실제 효과가 있어도 탐지하지 못한다(제2종 오류). 반대로 표본 수 기준 없이 "유의해질 때까지" 테스트를 이어가면 거짓 양성(제1종 오류)이 급증한다.

통계학자 Jacob Cohen은 저서 *Statistical Power Analysis for the Behavioral Sciences*(1988)에서 검정력(power) 80%를 표준으로 제안했다(1988년 기준, [CXL 해설](https://cxl.com/blog/statistical-power/) 참조). 검정력 80%란 실제 효과가 존재할 때 이를 올바르게 탐지할 확률이 80%라는 뜻이다. 나머지 20%는 효과가 있는데도 놓치는 위험(β 오류)이다.

---

## 표본 크기 계산 공식

두 비율(전환율)을 비교하는 A/B테스트의 표본 크기 공식은 다음과 같다([Evan Miller 표본 크기 계산기](https://www.evanmiller.org/ab-testing/sample-size.html)).

```
n = (Z_α/2 + Z_β)² × [p₁(1-p₁) + p₂(1-p₂)] / (p₂ - p₁)²
```

| 기호 | 의미 | 일반적 설정 |
|------|------|-------------|
| α | 유의수준 | 0.05 (5%) |
| β | 제2종 오류 허용률 | 0.20 (검정력 80%) |
| Z_α/2 | 유의수준의 Z값 | 1.96 |
| Z_β | 검정력의 Z값 | 0.84 |
| p₁ | 대조군 전환율(기준선) | 측정값 사용 |
| p₂ | 실험군 기대 전환율 | p₁ + MDE |
| MDE | 최소 탐지 효과 | 비즈니스 결정 |

**계산 예시**: 현재 전환율 10%(p₁=0.10), 5%p 상승을 탐지하고 싶다면(p₂=0.15), 유의수준 5%, 검정력 80% 기준으로 **그룹당 약 686명**이 필요하다. 양쪽 합치면 1,372명이다.

핵심은 MDE(최소 탐지 효과)다. MDE를 2%p로 줄이면 필요 표본은 약 4,000명 이상으로 급증한다. 작은 차이를 잡으려면 그만큼 많은 데이터가 필요하다.

## 실무에서 자주 틀리는 5가지

### 실수 1: 표본 크기를 사전에 계산하지 않는다

실무에서 가장 흔한 패턴이 "일단 돌리고 보자"다. 사전 계산 없이 시작하면 테스트 기간이 불분명해지고, 결과를 언제 확인해야 할지 기준이 없다.

**해결**: 테스트 시작 전에 기준선 전환율, MDE, 유의수준, 검정력을 정하고 필요 표본 수를 산출한다. [Evan Miller 계산기](https://www.evanmiller.org/ab-testing/sample-size.html)나 [Optimizely 계산기](https://www.optimizely.com/sample-size-calculator/)를 활용하면 30초면 된다.

### 실수 2: 중간에 결과를 반복 확인하고 조기 종료한다

"유의하게 나왔다"는 순간 테스트를 멈추는 관행이다. 이를 피킹(peeking) 문제라고 한다. Evan Miller의 분석에 따르면, 매일 결과를 확인하면서 p<0.05가 될 때 중단하면 **실제 거짓 양성률이 5%가 아니라 30% 이상**으로 올라간다([How Not To Run an A/B Test, Evan Miller](https://www.evanmiller.org/how-not-to-run-an-ab-test.html), 2015년 기준).

**해결 방법 2가지**:
- **고정 표본법**: 사전에 계산한 표본 수에 도달할 때까지 결과를 확인하지 않는다
- **순차 검정(sequential testing)**: 중간 확인이 필요하다면 alpha spending function이나 mSPRT(혼합 순차 확률비 검정)를 적용한다. Statsig 등 실험 플랫폼이 이 방식을 지원한다([Statsig 순차 검정 해설](https://www.statsig.com/perspectives/sequential-testing-ab-peek))

### 실수 3: 다중 비교 보정을 하지 않는다

하나의 테스트에서 3개 변형(A, B, C)을 동시에 비교하면 거짓 양성 확률이 올라간다. 5개 변형을 테스트할 경우 최소 1개에서 거짓 양성이 나올 확률은 약 23%다([Statsig 다중 비교 보정](https://www.statsig.com/blog/multiple-comparison-corrections-in-a-b)).

**해결**: Bonferroni 보정(α를 비교 횟수로 나눔)이 가장 단순하다. 3개 변형이면 유의수준을 0.05/3 ≈ 0.017로 설정한다. 더 정교한 방법으로는 Benjamini-Hochberg 절차가 있다.

### 실수 4: MDE를 비즈니스 맥락 없이 설정한다

"전환율 0.1%p 차이도 잡겠다"는 기술적으로 가능하지만, 수십만 명의 표본이 필요하다. 반대로 "20%p 차이만 보겠다"는 의미 있는 효과를 놓친다.

**해결**: MDE는 비즈니스 의사결정 기준으로 정한다. "전환율이 최소 몇 %p 올라야 이 기능을 출시할 가치가 있는가?"라는 질문에 답하면 MDE가 나온다. 실제로 직접 계산기를 돌려보면, 기준선 전환율의 10~20% 상대적 변화를 MDE로 설정했을 때 현실적인 표본 수가 나온다([SplitMetrics 표본 크기 가이드](https://splitmetrics.com/blog/mobile-a-b-testing-sample-size/)).

### 실수 5: 기준선 전환율을 잘못 추정한다

표본 크기 공식에서 기준선 전환율(p₁)은 핵심 입력값이다. 과거 데이터 없이 "대충 5%쯤 되지 않을까"로 넣으면 계산 결과가 왜곡된다. 기준선이 50%에 가까울수록 분산이 커지므로 필요 표본 수가 달라진다.

**해결**: 최소 2~4주간의 과거 데이터에서 기준선 전환율을 측정한다. 세그먼트별 차이가 크면(모바일 vs 데스크톱 등) 세그먼트를 분리해서 계산한다.

## 실무 체크리스트

테스트 설계 시 아래 항목을 순서대로 확인한다.

| 단계 | 체크 항목 | 비고 |
|------|-----------|------|
| 1 | 기준선 전환율 측정 | 최소 2주 데이터 |
| 2 | MDE 결정 | 비즈니스 의사결정 기준 |
| 3 | 유의수준·검정력 설정 | 통상 α=0.05, power=0.80 |
| 4 | 표본 크기 산출 | 공식 또는 온라인 계산기 |
| 5 | 테스트 기간 추정 | 일일 트래픽 ÷ 필요 표본 |
| 6 | 중간 확인 규칙 수립 | 고정 표본법 or 순차 검정 |
| 7 | 다중 비교 보정 적용 | 변형 2개 이상일 때 |

## 우아한형제들 B마트의 실험문화 사례

한국 기업 중 A/B테스트 문화를 실험 0건에서 출발해 단계별로 구축한 사례로 우아한형제들의 B마트가 있다([우아한형제들 기술블로그](https://techblog.woowahan.com/13726/)). 이 과정에서 강조한 원칙이 3가지다.

1. **실패를 받아들이는 전제**: 실험 결과가 부정적이어도 학습으로 인정한다
2. **지표 성공 여부보다 학습 중심**: 가설이 틀렸다는 것도 유효한 결과다
3. **실험 영향에 대한 투명성**: 결과를 조직 전체에 공유한다

표본 크기 설계는 이런 문화의 기술적 토대다. 표본 설계가 허술하면 "학습"이 아니라 "노이즈에서 패턴을 읽는 착각"이 된다.

## 정리

A/B테스트에서 표본 크기는 실험의 신뢰도를 결정하는 기본 설계 요소다. 사전 계산 없이 시작하거나, 중간에 결과를 반복 확인하거나, 다중 비교를 보정하지 않으면 결과 자체가 신뢰할 수 없게 된다.

테스트를 시작하기 전에 기준선 전환율을 측정하고, 비즈니스 맥락에서 MDE를 정하고, 공식으로 필요 표본 수를 산출하는 것이 첫 단계다. 이 과정을 건너뛰면 테스트를 아무리 많이 돌려도 의사결정 근거가 되지 못한다. 다음 A/B테스트를 설계할 때 [Evan Miller 계산기](https://www.evanmiller.org/ab-testing/sample-size.html)에 기준선 전환율과 MDE를 넣어보는 것부터 시작한다.
